# aura-privesc

Salesforce Aura/Lightning privilege escalation scanner. Discovers exposed Aura endpoints, enumerates object-level CRUD permissions, tests Apex controllers, and optionally proves record-level access through interactive CRUD validation.

## Install

Requires Python 3.11+.

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -e .
```

Or install dependencies directly:

```bash
python3 -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
```

## Usage

### Basic scan (guest mode)

```bash
aura-privesc -u https://target.my.site.com
```

### With proxy (e.g. Burp Suite)

```bash
aura-privesc -u https://target.my.site.com --proxy http://127.0.0.1:8080 --insecure
```

### Interactive CRUD validation

After the initial scan, walks you through per-object read/create/update/delete testing with confirmation prompts:

```bash
aura-privesc -u https://target.my.site.com --interactive
```

### Authenticated scan

Supply an Aura token captured from a browser session:

```bash
aura-privesc -u https://target.my.site.com -t "eyJhbGciOi..."
```

### JSON output

```bash
aura-privesc -u https://target.my.site.com --json
```

### Custom object/Apex lists

```bash
aura-privesc -u https://target.my.site.com --objects-file custom_objects.txt --apex-file apex_methods.txt
```

Files should contain one entry per line (`#` comments and blank lines are ignored).

## Scan Phases

| Phase | Description |
|-------|-------------|
| 1 | **Discovery** — probes endpoint paths, extracts fwuid and aura.context from community HTML |
| 2 | **User context** — identifies current user, checks SOQL capability, discovers config objects |
| 3 | **Object enumeration** — checks CRUD permissions via `getObjectInfo`, counts records via `getItems`, validates findings to filter false positives |
| 4 | **Apex testing** — discovers controllers from JS files, tests each method, classifies as CALLABLE/DENIED/NOT_FOUND |
| 5 | **Interactive CRUD validation** (opt-in) — per-object read/create/update/delete with user prompts |
| 6 | **Reporting** — generates an HTML report with all findings, proofs, and risk ratings |

## Output

- **Terminal** — Rich-formatted tables with CRUD permission flags, risk badges, and proof curl commands
- **JSON** (`--json`) — machine-readable scan results on stdout
- **HTML report** — self-contained file with embedded CSS (dark theme), generated by default in the current directory. Use `--no-report` to suppress or `--report-dir ./reports` to change location

## Risk Classification

Objects are classified based on sensitivity and permissions:

| Level | Criteria |
|-------|----------|
| CRITICAL | Critical object (User, Account, Contact, PermissionSet, etc.) with write permissions |
| HIGH | Critical object read-only, or sensitive object with write permissions |
| MEDIUM | Sensitive object read-only, or any object with write permissions |
| LOW | Non-sensitive object, read-only access |

## Options

```
-u, --url TEXT          Target Salesforce base/community URL (required)
-t, --token TEXT        Aura token (omit for guest mode)
-i, --interactive       Interactive CRUD validation mode
--json                  Output as JSON (incompatible with --interactive)
--report / --no-report  Generate HTML report (default: on)
--report-dir DIRECTORY  Directory for HTML report output
--objects-file PATH     File with additional object API names
--apex-file PATH        File with Apex controller.method pairs
--context TEXT          Manual aura.context JSON
--endpoint TEXT         Manual aura endpoint path
--skip-crud             Skip per-object CRUD permission checks
--skip-records          Skip record count enumeration
--skip-apex             Skip Apex controller testing
--skip-validation       Skip finding validation (faster, may include false positives)
--timeout INTEGER       HTTP timeout in seconds (default: 30)
--delay INTEGER         Delay between requests in ms (default: 0)
--concurrency INTEGER   Max concurrent requests (default: 5)
--proxy TEXT            HTTP proxy URL
--insecure              Disable TLS verification
-v, --verbose           Show raw request/response data
```

## Disclaimer

This tool is intended for authorized security testing only. Only use it against systems you have explicit permission to test.
